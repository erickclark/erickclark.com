What I've been up to lately: Infoblox

For the last three and a half years I've worked at Infoblox, where at different times I've owned the UX of three of the company's products. I've worked closely with product managers, software engineers, SMEs and other designers to define UX requirements, create wireframes and design mockups, run user tests and evaluate the analytics of features post launch.

The first product I worked on at Infoblox was a SASE product for network engineers (Wikipedia can explain SASE better than I can) that was unfortunately canceled when new management came into the company. The second product was BloxOne DDI, a product that allows network administrators to configure and manage DNS servers and the domain names served by them. It also allows network administrators to manage and configure DHCP servers, which automatically assign IP addresses to machines on a network. BloxOne DDI also allows users to organize and track those assigned IP addresses.

My work on BloxOne DDI included designing small updates that added new options to existing configuration interfaces. I also designed large features, such as the UX for configuring and monitoring DNS Load Balancers, which allow internet traffic to be routed to several different servers based on conditions such as location or server status. For example, because facebook.com is served from many web servers all over the world, a DNS load balancer can route your web request to the nearest server with the least amount of internet traffic.

The third product I've worked on is a network asset inventory system that allows network administrators to track devices on their on-premises network as well as virtual devices in the cloud. This is a new product that I've been working on since its inception. I've collaborated with product managers to write user cases, define features and iteratively design the user experience. The network asset inventory system allows network administrators to import asset data for both on-premises and virtual devices and use that information to identify problematic devices (i.e., that may have been infected with malware) or viewing aggregate data (i.e., a list of devices that are running an out of date operating systems, like Windows 7). This information allows network administrators to find and troubleshoot devices on their network and gain insight into their network's attack surface.
